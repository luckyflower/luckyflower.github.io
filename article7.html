HTML5 history技术

首先先来看下它的API：

| API | 描述| | :---- |----: | |history.pushState(data, title [, url])|往历史记录堆栈顶部添加一条记录；data会在onpopstate事件触发时作为参数传递过去；title为页面标题，当前所有浏览器都会忽略此参数；url为页面地址，可选，缺省为当前页地址。 |history.replaceState(data, title [, url])；|更改当前的历史记录，参数同上。 |history.state|用于存储以上方法的data数据，不同浏览器的读写权限不一样。 |window.onpopstate|响应pushState或replaceState的调用；

检测浏览器是否支持History API的代码：

function supports_history_api(){
    return !!(window.history && history.pushState);
}

为什么要使用History API 通过其我们可以实现页面的无刷新跳转。
